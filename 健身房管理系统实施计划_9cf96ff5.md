# 健身房管理系统实施计划

## 技术栈确认

- **后端**: SpringBoot 2.7.x + MyBatis-Plus + MySQL 8.0 + Spring Security + Maven
- **前端会员端**: Vue3 + Element Plus + Axios + Vue Router + Pinia + Video.js + ECharts
- **前端管理端**: Vue3 + Element Plus（独立应用或同一应用不同路由模块）
- **工具**: Lombok, Hutool, PageHelper

## 数据库设计（11张表）

全部表在 Task 1 中通过 SQL 脚本一次性创建：
1. `sys_user` - 管理员/教练用户表
2. `member` - 会员表
3. `course_category` - 教程分类表
4. `fitness_course` - 健身教程表
5. `course_interact` - 教程互动表(点赞/收藏/评论)
6. `coach` - 教练表
7. `gym_class` - 健身课程表
8. `class_reserve` - 课程预约表
9. `gym_equipment` - 器材表
10. `announcement` - 公告/资讯表
11. `orders` - 订单表

## Task 1: 后端项目骨架搭建

**目标**: 创建 SpringBoot Maven 项目，配置所有依赖，创建数据库初始化脚本，搭建基础框架代码。

**具体内容**:
- 创建 `gym-server` Maven 项目，pom.xml 包含所有依赖（SpringBoot, MyBatis-Plus, MySQL, Spring Security, Lombok, Hutool, PageHelper）
- `application.yml`: 数据库连接、服务端口(8080)、MyBatis-Plus配置、文件上传配置
- 通用类: `Result<T>` 统一响应、`PageResult` 分页响应、全局异常处理器 `GlobalExceptionHandler`、`BusinessException` 自定义异常
- MyBatis-Plus 配置: 分页插件、自动填充（createTime/updateTime）
- CORS 跨域配置
- SQL 初始化脚本: 创建 `gym_db` 数据库及全部 11 张表，插入基础测试数据（管理员账号、教程分类、示例数据）
- 全部 11 个 Entity 实体类（使用 Lombok + MyBatis-Plus 注解）

**产出**: 可启动的 SpringBoot 空壳项目 + 完整数据库脚本 + 全部实体类

## Task 2: Spring Security + JWT 认证模块

**目标**: 实现完整的认证授权体系，支持管理员/教练/会员三端登录。

**具体内容**:
- JWT 工具类: 生成/解析/验证 Token
- `JwtAuthenticationFilter`: 请求拦截、Token校验
- Spring Security 配置: 路径权限规则、密码编码器、认证管理器
- 管理员登录接口 `POST /api/admin/login`
- 会员登录接口 `POST /api/front/login`
- 会员注册接口 `POST /api/front/register`
- 获取当前用户信息接口
- 对应的 Service、Mapper 层代码

**产出**: 完整的登录/注册/鉴权链路

## 注意事项

- 数据库连接信息默认: localhost:3306, 用户名 root, 密码需用户确认
- 文件上传暂用本地存储（可后续扩展OSS）
- 视频播放使用外部链接或本地文件，不做流媒体处理
- Spring Security 登录鉴权使用 JWT 无状态方案
- 前端管理端和会员端为两个独立 Vue3 项目，共享后端 API
